Podstawowa struktura:
```kotlin

```

## Dodajemy `Checkbox`y

### Pierwszy `Checkbox` z napisem "Programowaniem"
```kotlin
 Column() {
        Text(text="Zainteresowania")

            Checkbox(
                checked = true,
                onCheckedChange = { })
            Text(text="Programowanie")
}
```
Napis "Programowanie znalazł się pod `Checkbox`em.
Byłoby lepiej, gdyby znajdował się po prawej stronie tego napisu.
Dlatego też dodamy `Row`.
```kotlin
Column() {
        Text(text="Zainteresowania")

        Row(verticalAlignment = Alignment.CenterVertically){
            Checkbox(
                checked = true,
                onCheckedChange = { })
            Text(text="Programowanie")
        }
}
```
:::tip `Row` *vertical* a jednak *horizontal*
- `Row` elementy są ustawiane horyzontalnie (w poziomie) – od lewej do prawej:
    - oś główna (*main axis*) jest pozioma
    - oś pomocnicza (*cross axis*) jest pionowa (*vertical*)
- `verticalAlignment:
    - oznacza jak elementy mają być wyrównane w pionie względem siebie
    - w `Row` nie ma nic wspólnego z układem lewo–prawo
    - to tylko pionowe wyrównanie elementów, które i tak są już obok siebie.

:::

### Logika działania naszego `Checkbox`a
- potrzebujemy zmiennej, która będzie przechowywać informacje o tym, czy checkbox jest zaznaczony
- potrzebujemy mechanizmu, który umożliwi zapamiętanie tej zmiennej

```kotlin
 var isProgrammingChecked by remember { mutableStateOf(false) }

    Column() {
        Text(text="Zainteresowania")

        Row(verticalAlignment = Alignment.CenterVertically){
            Checkbox(
                checked = isProgrammingChecked,
                onCheckedChange = { newValue ->
                    isProgrammingChecked = newValue
                    Log.i("check", "$isProgrammingChecked")
                })
            Text(text="Programowanie")
        }

    }
```
**Wyjaśnienie**
` var isProgrammingChecked by remember { mutableStateOf(false) }` znaczy:
    - stworzy zmienną, która będzie pamiętać, czy użytkownik zaznaczył `checkbox` "Programowanie",
    - na początku niech będzie odznaczona (`false`).
    - nawet jak ekran się przeładuje (np. obrócisz telefon), to ta informacja ma nie zniknąć.
- `remember`
    - „Zapamiętaj to dla mnie”.
    - Gdyby go nie było:
        - za każdym razem, gdy ekran się odświeży (np. obrócisz telefon albo Compose coś przerysuje),
        - stan wróciłby do początkowego i checkbox „zapomniałby”, że był zaznaczony.
- `mutableStateOf`
    - tworzy specjalny obiekt, który „pilnuje” wartości `false` (czyli checkbox odznaczony)
    - gdy użytkownik kliknie `checkbox`, wartość zmieni się na `true`
    - a Compose automatycznie przerysuje ekran
    - to specjalny obiekt, który potrafi powiedzieć Compose’owi:
        - „Hej! Ja się właśnie zmieniłem! Proszę przerysuj ekran!”

## Dodajemy kolejne dwa `Checkbox`y - "Sport" i "Muzyka"
- zwróć uwagę, że `Column` dostał `modifier`

```kotlin
@Composable
fun UserSettings( modifier: Modifier = Modifier) {

    var isProgrammingChecked by remember { mutableStateOf(false) }
    var isSportChecked by remember { mutableStateOf(false) }
    var isMusicChecked by remember { mutableStateOf(false) }

    Column( modifier = modifier) {
        Text(text="Zainteresowania",
            fontSize = 20.sp,
            modifier = Modifier.padding(top=24.dp, bottom = 10.dp)
        )

        Row(verticalAlignment = Alignment.CenterVertically){
            Checkbox(
                checked = isProgrammingChecked,
                onCheckedChange = { newValue ->
                    isProgrammingChecked = newValue
                })
            Text(text="Programowanie")
        }

        Row(verticalAlignment = Alignment.CenterVertically){
            Checkbox(
                checked = isSportChecked,
                onCheckedChange = { newValue ->
                    isSportChecked = newValue
                })
            Text(text="Sport")
        }

        Row(verticalAlignment = Alignment.CenterVertically){
            Checkbox(
                checked = isMusicChecked,
                onCheckedChange = { newValue ->
                    isMusicChecked = newValue
                })
            Text(text="Muzyka")
        }
    }
}
```

## Dynamiczne tworzenie `Checkbox`ów
- mamy kilka `Checkbox`ów, więc nasza funkcja `Composable` zaczyna być nieczytelna
- będziemy dynamicznie generować `Checkbox`y



### `data class` - model dla zainteresowań

`Interest.kt`
```

data class Interest (val name: String, var checked: Boolean)

```

### Utworzenie listy obiektów `Interest`
Zamiast oddzielnych zmiennych w naszej funkcji `Composable` utworzymy listę obiektów `Interest`:
```kotlin
@Composable
fun UserSettings( modifier: Modifier = Modifier) {
    // var isProgrammingChecked by remember { mutableStateOf(false) }
    // var isSportChecked by remember { mutableStateOf(false) }
    // var isMusicChecked by remember { mutableStateOf(false) }

    val interests = remember {
    mutableStateListOf(
        Interest("Programowanie", false),
        Interest("Muzyka", false),
        Interest("Sport", false)
    )
}
```
### Dynamiczne generowanie `Checkbox`ów
```kotlin
@Composable
fun UserSettings( modifier: Modifier = Modifier) {
//...

 Column( modifier = modifier) {
        Text(text="Zainteresowania",
            fontSize = 20.sp,
            modifier = Modifier.padding(top=24.dp, bottom = 10.dp)
        )

        interests.forEachIndexed { index, interest ->
            Row(verticalAlignment = Alignment.CenterVertically){
                Checkbox(
                    checked = interest.checked,
                    onCheckedChange = {

                        // copy stworzy nowy obiekt
                        // więc zmieni się lista
                        // więc UI zostanie poinformowane o zmiania
                        interests[index] = interest.copy(checked = it)

                        /*ta list obiektów jest reaktywna (informuje UI o zmianach
                        * sam obiekt i jego właściwości nie są reaktywne, więc
                        * nie informują UI o swoich zmianach
                        * */
                       // interest.checked = it
                    }
                )
                Text(text=interest.name)
            }
        }
    }
```


## `remember` vs `rememberDaveable`
W Jetpack Compose mamy dwa podstawowe sposoby na to, aby UI "pamiętało", co się w nim dzieje. Wybór odpowiedniego zależy od tego, jak długo dane mają przetrwać.

| Cecha | `remember` | `rememberSaveable` |
| :--- | :--- | :--- |
| **Przetrwanie rekompozycji** (odświeżenie fragmentu UI) | ✅ Tak | ✅ Tak |
| **Przetrwanie rekonfiguracji** (np. obrót ekranu) | ❌ Nie | ✅ Tak |
| **Zamykanie aplikacji** (proces w tle) | ❌ Nie | ✅ Tak (częściowo) |
| **Wymagania systemowe** | Brak (pamięć RAM) | Dane muszą być możliwe do zapisania w `Bundle` |
| **Najlepszy dla...** | Animacji i stanów tymczasowych | Formularzy, checkboxów i danych użytkownika |

```kotlin
// ten kod
var isSportChecked by remember  { mutableStateOf(false)}
var isProgrammingChecked by remember  { mutableStateOf(false)}
var isMusicChecked by remember  { mutableStateOf(false)}

// zamienilibyśmy na
var isSportChecked by rememberSaveable  { mutableStateOf(false)}
var isProgrammingChecked by rememberSaveable  { mutableStateOf(false)}
var isMusicChecked by rememberSaveable  { mutableStateOf(false)}


```

:::tip Złota zasada
- Używaj `remember`, gdy masz dane tymczasowe, które nie są krytyczne
(np. animacja, która może zacząć się od nowa).
- Używaj `rememberSaveable` dla wszystkiego, co wpisał lub zaznaczył użytkownik
(np. tekst w polu wyszukiwania, zaznaczone checkboxy),
aby nie irytować go koniecznością ponownego wypełniania formularza po obróceniu telefonu.


:::
