Model danych​

Co to jest Model danych?

  • ta klasa opisuje strukturę danych, z jakimi pracuje dana aplikacja.

Włączenie bindingu​

Czym jest ViewBinding?

To mechanizm Androida, który automatycznie generuje klasy dla każdego layoutu XML. Dzięki temu możemy:

  • bezpiecznie odwoływać się do widoków przez ID (binding.editTextTitle)
  • unikać findViewById
  • mieć autouzupełnianie i sprawdzanie typów
  • w pliku buld.gradle.kts (Module) dodajemy:
// ...
    buildFeatures{
        viewBinding = true
    }
}

dependencies {
    implementation(libs.androidx.core.ktx)
    implementation //...

Metoda zapisu danych​

Do klasy MainActivity dodajemy nową metodę:

private fun saveMoviesToJsonFile(){
        val gson = Gson()
        val jsonString = gson.toJson(movieList) // zamienia listę na tekst w formacie JSON - SERIALIZACJA


        // Context.MODE_PRIVATE tylko ta aplikacja ma dostęp do tego pliku (bezpiecznie)
        // .use { output -> ... }
            // To sprytna konstrukcja z Kotlin: otwórz strumień, użyj go,
            // a potem go automatycznie zamknij (czyli close() zostanie wywołane automatycznie).
            // toByteArray() = „przetłumacz tekst na ciąg bajtów”
        openFileOutput(
            "movies.json",
            Context.MODE_PRIVATE).use{ output ->
                output.write(jsonString.toByteArray())
        }

        Toast.makeText(this,
            "Zapisano ${movieList.size} filmów to JSON",
            Toast.LENGTH_SHORT
            ).show()
    }

  • W Android Studio kliknij View -> Tool Windows -> Device File Explorer
