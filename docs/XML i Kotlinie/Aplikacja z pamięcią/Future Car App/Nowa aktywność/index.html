CarDetailsActivity

Utworzymy nową aktywność, w które będziemy wyświetlali wszystkie informacje o klikniętym samochodzie.

Zmiana modelu​

Do naszego modelu dodamy nowe właściwości: cena, czas realizacji zakupu.

import java.io.Serializable

data class FutureCar(
    val name: String,
    val imageUrl: String,
    val productionYear: Int,
    val topSpeed: Int,
    // Dodajemy nowe pola:
    val price: Int,
    val deliveryTime: String
) : Serializable // Pozwala na przesyłanie obiektu między oknami!

Modyfikacja fetchDataFromCloud​

Wewnątrz metody .addOnSuccessListener dodajemy:

for (document in documents) {
    val name = document.getString("name") ?: "Nieznany pojazd"
    val imageUrl = document.getString("imageUrl") ?: ""
    val year = document.getLong("productionYear")?.toInt() ?: 0
    val speed = document.getLong("topSpeed")?.toInt() ?: 0

    // NOWE POLA (jeśli ich nie ma w bazie, dajemy domyślne ceny i czas):
    val price = document.getLong("price")?.toInt() ?: 2500000
    val deliveryTime = document.getString("deliveryTime") ?: "12 miesięcy"

    // Tworzymy zaktualizowany obiekt
    val car = FutureCar(name, imageUrl, year, speed, price, deliveryTime)
    allCarsList.add(car)
}

Od CarDetailsActivity do MainActivity​

Teraz logika działania w CarDetailsActivity.

class CarDetailsActivity : AppCompatActivity() {
    private lateinit var binding: ActivityCarDetailsBinding

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        enableEdgeToEdge()
        // "Pompujemy" nasz piękny nowy wygląd z XML
        binding = ActivityCarDetailsBinding.inflate(layoutInflater)
        setContentView(binding.root)

        ViewCompat.setOnApplyWindowInsetsListener(binding.root) { v, insets ->
            val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars())
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom)
            insets
        }

        //  ODBIÓR DANYCH (Profesjonalna obsługa starych i nowych Androidów)
        val car = if (android.os.Build.VERSION.SDK_INT >= android.os.Build.VERSION_CODES.TIRAMISU) {
            // Nowy, bezpieczny sposób dla Androida 13 i nowszych
            intent.getSerializableExtra("CAR_DATA", FutureCar::class.java)
        } else {
            // Stary sposób dla starszych telefonów
            @Suppress("DEPRECATION") // Mówimy kompilatorowi: "Wiem, co robię, przestań narzekać"
            intent.getSerializableExtra("CAR_DATA") as? FutureCar
        }

        //  Sprawdzamy, czy paczka dotarła bezpiecznie
        if (car != null) {
            // Wstrzykujemy dane do interfejsu
            binding.detailNameTextView.text = car.name
            binding.detailYearTextView.text = "Rok produkcji: ${car.productionYear}"
            binding.detailSpeedTextView.text = "Prędkość max: ${car.topSpeed} km/h"
            // Formatuje cenę - taki mały detal dla lepszego wyglądu
            binding.detailPriceTextView.text = "Cena: ${car.price} PLN"
            binding.detailDeliveryTextView.text = "Czas dostawy: ${car.deliveryTime}"

            // Zlecamy Malarzowi (Glide) pobranie wielkiego zdjęcia w pełnej krasie
            Glide.with(this)
                .load(car.imageUrl)
                .placeholder(R.mipmap.ic_launcher) // Zaślepka w trakcie ładowania z chmury
                .into(binding.detailImageView)

            // Ożywiamy nasz nowy przycisk powrotu!
            binding.backButton.setOnClickListener {
                finish() // To magiczne polecenie zamyka ekran detali i odsłania MainActivity
            }

        } else {
            // Zabezpieczenie: Gdyby paczka zginęła po drodze (np. błąd systemu)
            Toast.makeText(
                this,
                "Błąd ładowania danych pojazdu!",
                Toast.LENGTH_LONG).show()
            finish() // Zamykamy to okno i wracamy do listy
        }
    }
}

Czyszczenie SearchView​

Powrót do głównej aktywności będzie łączył się z wyczyszczeniem SearchViewa.

  • Kiedy w MainActivity klikamy w auto i uruchamiamy CarDetailsActivity, główny ekran wcale nie ginie!
  • Android go po prostu "usypia" w tle (przechodzi w stan onStop), żeby oszczędzać baterię.
  • Kiedy w detalach klikamy przycisk powrotu, MainActivity się budzi.
  • Ponieważ nigdy nie zostało zniszczone, wszystkie wpisane teksty i zmienne są dokładnie tam, gdzie je zostawiliśmy.

Chcielibyśmy, żeby powrót do MainActivity był powrotem do czystej aktywności głównej.

onResume()
  • Każdy ekran w Androidzie ma wbudowane specjalne metody reagujące na to, co się z nim dzieje.
  • Metoda onResume() uruchamia się zawsze, gdy ekran staje się w pełni widoczny dla użytkownika (zarówno przy pierwszym uruchomieniu aplikacji, jak i przy powrocie z innego okna!).
  • To idealne miejsce na nasze małe sprzątanie!

MainActivity jako jedna z metod w klasie (czyli poza onCreate!):

    override fun onResume() {
        super.onResume()

        // 1. Czyścimy pole wyszukiwarki.
        // Pierwszy parametr to nowy tekst (pusty), a drugi (false) mówi: "Nie zatwierdzaj tego automatycznie lupą"
        binding.carsSearchView.setQuery("", false)

        // 2. Zabieramy "skupienie" (focus) z wyszukiwarki.
        // Dzięki temu po powrocie klawiatura nie wyskoczy od razu sama na ekran!
        binding.carsSearchView.clearFocus()
    }
