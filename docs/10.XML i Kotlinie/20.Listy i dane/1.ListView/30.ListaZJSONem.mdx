# Lista i lokalny CRUD (JSON, SharedPreferences)
- Zbudujemy aplikację *MiniKino* z osobistą listą filmów.
- Do nasze listy będzie można dodać nowe filmy.
- Aplikacja zapisze je w pliku lokalnym.


Utwórzmy nowy projekt `Empty Views Activity` *MiniKino*.

## Model danych

:::tip Co to jest Model danych?
- to klasa (najczęściej data class w Kotlinie), która
- ta klasa opisuje strukturę danych, z jakimi pracuje dana aplikacja.

:::

- utwórz folder/package w folderze, w którym jest `MainActiviry`
- w tym nowym folderze zrób plik `MovieEntry`
```kotlin
data class MovieEntry(
    val title: String,
    val genre: String,
    val rating: Int
)
```

## `kotlinx.serialization` - zapisywanie danych do pliku
:::note Czym jest *serialization* (serializacja)?
To proces zamiany obiektów programu na zapis tekstowy lub binarny, tak aby można je było:
- zapisać do pliku,
- przesłać przez sieć,
- odtworzyć później w tej samej lub innej aplikacji.

:::

:::tip *Deserialization*
to proces odtworzenia obiektów programu z zapisanego formatu


:::

#### Metoda zapisu danych

:::tip `Json`
- ma dwie główne funkcje:
    - `encodeToString`  bierze obiekt Kotlina i "wypycha" go do Stringa (JSON).
    - `decodeFromString` bierze Stringa (JSON) i "wciąga" go do obiektu Kotlina.

- może występować w dwóch wariantach:
    - domyślny `Json` tłumacz, który tłumaczy wszystko "jednym ciągiem",
        bez przerw, żeby było szybciej.
        - np. `val tekst = Json.encodeToString(mojeDane)`
    - unikalna instancja
        - np. taka, która będzie formatować tekst ` val json = Json { prettyPrint = true }`
        - np. ignoruje pola niezgodne z modelem `val json = Json {
      ignoreUnknownKeys = true // "Ignoruj klucze, których nie znam"
      }`
:::


:::danger `@Serializable`
Instancja `Json` potrafi serializować tylko te klasy, które mają adnotację `@Serializable`

:::


Dopiszmy adnotację do naszego modelu
```kotlin
@Serializable
data class MovieEntry(
    val title: String,
    val genre: String,
    val rating: Int
)
```




Do klasy `MainActivity` dodajemy nową metodę:
```kotlin
    private fun saveMoviesToJsonFile() {
        try {
            val json = Json { prettyPrint = true }
            val jsonString = json.encodeToString(movieList)

            val file = File(filesDir, "movies.json")
            // filesDir:
            // - obiekt typu File
            // - /data/data/<nazwa.pakietu>/files/
            // - jest właściwością dziedziczoną po klasie Context
            file.writeText(jsonString)


            Toast.makeText(
                this,
                "Zapisano ${movieList.size} filmów to JSON",
                Toast.LENGTH_SHORT
            ).show()

        } catch (e: Exception) {

            Toast.makeText(
                this,
                "Błąd zapisu pliku!",
                Toast.LENGTH_SHORT
            ).show()

            e.printStackTrace() // dokładne błędy w Logcat
        }

    }
```

Dzięki temu lista będzie dostępna w nowej metodzie, która będzie odczytywała dane z JSON.


:::tip Trzy różne role, trzy różne odpowiedzialności
- Model danych (np. `movieList`):
    - przechowuje prawdziwe dane aplikacji
    - odpowiada strukturze biznesowej (tytuł, gatunek, ocena)
    - jest zapisywany / odczytywany z JSON
    - UI go nie wyświetla bezpośrednio


- Lista dla UI (np. `movieTitles`):
    - zawiera tekstową reprezentację danych
    - jest przygotowana specjalnie do wyświetlania
    - powstaje na podstawie modelu danych
    - to ją obserwuje adapter


- Adapter (`ArrayAdapter`)
    - łączy listę UI z `ListView`
    - nie zna modelu danych
    - reaguje tylko na zmiany tej jednej listy, którą dostał
    - wymaga wywołania `notifyDataSetChanged()`
:::


###### Definiujemy metodę do odczytu danych
Do klasy `MainActivity` dodamy nową metodę\
```kotlin
private fun loadMoviesFromJsonFile() {
        try {
            val file = File(filesDir, "movies.json")
            if (!file.exists()) return


            val jsonString = file.readText()
            val json = Json { ignoreUnknownKeys = true }
            val loadedList = json.decodeFromString<List<MovieEntry>>(jsonString)

            movieList.clear()
            movieList.addAll(loadedList)


            // lista używana przez adapter
            movieTitles.clear()
            movieTitles.addAll(
                movieList.map {
                    "${it.title} (${it.genre}) ★${it.rating}"
                }
            )
            listAdapter.notifyDataSetChanged()

        } catch (e: java.lang.Exception) {
            Toast.makeText(
                this,
                "Błąd odczytu pliku!",
                Toast.LENGTH_SHORT
            ).show()
            e.printStackTrace()
        }
    }
```

##### Wywołujemy metodę `loadMoviesFromJsonFile`
W metodzie `onCreate()` na końcu wywołujemy metodę `loadMoviesFromJsonFile`

## SharedPreferences
Cel:
- w będziemy zapisywali w informację o tym, jaki gatunek filmu użytkownik ostatnio wybrał
- gdy aplikacja zostanie uruchomina ponowanie, domyślnie zaznaczony będzie ostatnio wybrany gatunek


:::note SharedPreferences
- najprostszy sposób na trwałe przechowywanie małych ilości danych.
- Wyobraź sobie to jako system "klucz-wartość" (jak słownik lub mapa),
- takie dane klucz-wartość zostają zapisane w pamięci telefonu.

:::

### Nowe właściwości w `MainActivity.kt`
```kotlin
class MainActivity : AppCompatActivity() {
/...

    private val PREFS_NAME = "minikino_prefs"
    private val KEY_LAST_GENRE = "last_genre"

    override fun onCreate(savedInstanceState: Bundle?) {//...
```
- `PREFS_NAME`
    - wartością jest nazwa pliku
    - plik zostanie stworzony, jeżeli go jeszcze nie ma
    - w pliku zostaną zapisane dane
- `KEY_LAST_GENRE`
    - w tej właściwości przechowujemy nazwę klucza
    - pod tym kluczem zapiszemy dane

### Zmianny w ` binding.addButton.setOnClickListener ` (dodawaniu do listy)
Gdy tworzony jest nowy film, zapamiętany zostanie jego gatunek
```kotlin
 binding.addButton.setOnClickListener {
            val title = binding.titleEditText.text.toString()
            val genre = binding.genreSpinner.selectedItem.toString()
            val rating = binding.ratingBar.rating.toInt()


            val prefs = getSharedPreferences(PREFS_NAME, MODE_PRIVATE)
            prefs.edit {
                putString(KEY_LAST_GENRE, genre)
            }
//...
```
`getSharedPreferences(PREFS_NAME, MODE_PRIVATE)`
- otwiera (lub tworzy) zestaw preferencji aplikacji o podanej nazwie
- i zwraca obiekt,
- przez który możemy zapisywać i odczytywać ustawienia.
    - `prefs.getString(...)`
    - `prefs.getInt(...)`
    - `prefs.getBoolean(...)`
    - `prefs.edit().putString(...)`

- `prefs.edit { putString(KEY_LAST_GENRE, genre) }`
    - otwiera edycję `SharedPreferences`,
    - zapisuje parę `klucz-> wartość` i automatycznie zapisuje zmiany.
    - `prefs.edit { ... }`
        - tworzy obiekt `SharedPreferences.Editor`
        - Kotlin przekazuje go do funkcji strzałkowej `{ ... }`
        - Kotlin po wyjściu z bloku automatycznie wywołuje `apply()`

- `putString(KEY_LAST_GENRE, genre)` zapisuje:
    - klucz: `"last_genre"`
    - wartość: np. `"Sci-Fi"`



### Gdzie znajdziemy plik z zapisanymi danymi?
```
/data
 └── data
     └── <nazwa twojej aplikacji>

```


### Wczytywanie danych przy starcie aplikacji
W metodzie `onCreate` dodajemy :
```kotlin
    val prefs = getSharedPreferences(PREFS_NAME, MODE_PRIVATE)
    val lastGenre = prefs.getString(KEY_LAST_GENRE, null)
    lastGenre?.let {
        val index = genres.indexOf(it)
        if (index >= 0) {
            binding.genreSpinner.setSelection(index)
        }
    }
```
- `val prefs = getSharedPreferences(PREFS_NAME, MODE_PRIVATE)`
    - jak poprzednio tworzymy obiekt do obsługi pliku
- `prefs.getString(KEY_LAST_GENRE, null)`:
    - odczytujemy dane z pliku przy użyciu klucza
- `genres` to lista stringów z nazwami rodzajów filmów
- `lastGenre?.let { ... }`
    - `?` dalszy kod wykona się, o ile `lastGenre` nie jest `null`
    - `let` wykona zawartość bloku, przekazując w zmiennej `it` wartość zmiennej `lastGenre`
-`genres.indexOf(it)`
    - zwraca indeks wartości zapisanej w `it`
    - u nas `it` to `lastGenre`, czyli string ostatnio wybranego rodzaju filmu
