# Lista i lokalny CRUD (JSON, SharedPreferences)
- Zbudujemy aplikację *MiniKino* z osobistą listą filmów.
- Do nasze listy będzie można dodać nowe filmy.
- Aplikacja zapisze je w pliku lokalnym.


Utwórzmy nowy projekt `Empty Views Activity` *MiniKino*.

## Model danych

:::tip Co to jest Model danych?
- to klasa (najczęściej data class w Kotlinie), która
- ta klasa opisuje strukturę danych, z jakimi pracuje dana aplikacja.

:::

- utwórz folder/package w folderze, w którym jest `MainActiviry`
- w tym nowym folderze zrób plik `MovieEntry`
```kotlin
data class MovieEntry(
    val title: String,
    val genre: String,
    val rating: Int
)
```

## `kotlinx.serialization` - zapisywanie danych do pliku
:::note Czym jest *serialization* (serializacja)?
To proces zamiany obiektów programu na zapis tekstowy lub binarny, tak aby można je było:
- zapisać do pliku,
- przesłać przez sieć,
- odtworzyć później w tej samej lub innej aplikacji.

:::

:::tip *Deserialization*
to proces odtworzenia obiektów programu z zapisanego formatu


:::



### Dodajemy bibliotekę `serialization` do aplikacji

W `.toml` dodajemy i synchronizujemy:


#### Metoda zapisu danych

:::tip `Json`
- ma dwie główne funkcje:
    - `encodeToString`  bierze obiekt Kotlina i "wypycha" go do Stringa (JSON).
    - `decodeFromString` bierze Stringa (JSON) i "wciąga" go do obiektu Kotlina.

- może występować w dwóch wariantach:
    - domyślny `Json` tłumacz, który tłumaczy wszystko "jednym ciągiem",
        bez przerw, żeby było szybciej.
        - np. `val tekst = Json.encodeToString(mojeDane)`
    - unikalna instancja
        - np. taka, która będzie formatować tekst ` val json = Json { prettyPrint = true }`
        - np. ignoruje pola niezgodne z modelem `val json = Json {
      ignoreUnknownKeys = true // "Ignoruj klucze, których nie znam"
      }`
:::


:::danger `@Serializable`
Instancja `Json` potrafi serializować tylko te klasy, które mają adnotację `@Serializable`

:::


Dopiszmy adnotację do naszego modelu
```kotlin
@Serializable
data class MovieEntry(
    val title: String,
    val genre: String,
    val rating: Int
)
```




Do klasy `MainActivity` dodajemy nową metodę:
```kotlin
    private fun saveMoviesToJsonFile() {
        try {
            val json = Json { prettyPrint = true }
            val jsonString = json.encodeToString(movieList)

            val file = File(filesDir, "movies.json")
            // filesDir:
            // - obiekt typu File
            // - /data/data/<nazwa.pakietu>/files/
            // - jest właściwością dziedziczoną po klasie Context
            file.writeText(jsonString)


            Toast.makeText(
                this,
                "Zapisano ${movieList.size} filmów to JSON",
                Toast.LENGTH_SHORT
            ).show()

        } catch (e: Exception) {

            Toast.makeText(
                this,
                "Błąd zapisu pliku!",
                Toast.LENGTH_SHORT
            ).show()

            e.printStackTrace() // dokładne błędy w Logcat
        }

    }
```

#### Gdzie jest ten plik?
- W Android Studio kliknij *View* -> *Tool Windows* -> *Device File Explorer*
- `data > data > <nazwa_projektu> > files` tu będzie nasz plik `movies.json`
